Project Intro

112 Gb/s Single-Ended Simultaneous Bi-Directional
Transceiver

A high-speed Die-to-Die / SerDes / CMOS IC design project that explored how to push more bandwidth through a single-ended link without losing timing margin or link stability.

I reproduced and integrated a single-ended simultaneous bi-directional (SBD) transceiver architecture, then verified the key TX, RX, and clocking blocks in simulation. The goal was to understand the architecture-level tradeoff between bandwidth density, complexity, and power. The final simulation flow showed a full-rate reconstructed output, quarter-rate receiver operation, and timing margin consistent with the target condition.

Why This Project

AI and HPC chiplets need die-to-die links that offer more bandwidth while keeping edge density high. That creates a hard tradeoff: if the interface gets too complicated, power and implementation cost rise quickly; if it stays too simple, the bandwidth per wire is not enough.

This project explores the value of single-ended simultaneous bi-directional signaling as one way to improve bandwidth density on short-reach die-to-die channels. In plain terms, it tries to send and receive on the same wire at the same time while still keeping the link usable at very high speed.

What I Built

Overall Architecture

Designed and reproduced the SBD transceiver architecture, including the TX path, RX path, and verification flow.

TX Path

Implemented the N-over-N TX driver and the replica path used for subtraction-based decoupling.

RX Front-End

Built the dynamic equalizer front-end to compensate channel loss and help recover the data stream at the receiver.

Data Recovery

Implemented the SR latch and PRBS generator needed for stable output capture and repeatable simulation input.

Quarter-Rate Verification

Constructed the quarter-rate, time-interleaved verification architecture with phase-shifted clocks and lane recombination.

Verification Flow

Ran transient, AC, eye, and bathtub-style verification to check functionality, timing margin, and channel compensation behavior.

System Architecture

The overall system has three main parts: the transmitter, the channel, and the receiver. The transmitter creates the coupled signal and a replica signal. The channel represents the single-ended die-to-die interconnect. The receiver uses a quarter-rate time-interleaved front-end so each lane can run at a lower effective speed while still supporting the full link rate.

Key Technical Ideas

Simultaneous Bidirectional Signaling

The same wire is used for both transmit and receive. That improves wiring efficiency, but it requires careful separation of the local TX contribution from the incoming signal.

Quarter-Rate Time Interleaving

Using four phase-shifted clock lanes reduces the per-lane operating speed, which makes the receiver easier to stabilize at the target data rate.

Dynamic Equalization

A dynamic equalizer helps recover high-frequency content lost in the channel so the receiver can still make a reliable decision.

Replica-Based Subtraction

A replica path is used to estimate the local TX effect, so the receiver can subtract or decouple the local swing before making a decision.

Verification & Results

The most important verification result was that the design reconstructed a full-rate output in simulation. I also checked the quarter-rate equalizer behavior directly, then used eye and bathtub analysis to confirm that the timing window was still usable at the target condition.

Challenges & Tradeoffs

A single equalizer stage was not stable enough at the target speed, so the design moved to a quarter-rate quadrature-clock architecture. That choice increased area and implementation complexity, but it improved stability and timing margin, which was the more important system-level goal.

This project is a good example of a classic high-speed design tradeoff: more structure and more clocking complexity can be worth it if the result is a receiver that actually works at the required speed.

Takeaways

  • Cadence and transistor-level simulation workflow: building and checking TX, RX, and clocking blocks in context.
  • Signal integrity and timing analysis: using eye and bathtub-style checks to judge whether the link is usable.
  • Equalization versus timing tradeoff analysis: understanding when to add complexity to recover margin.
  • System integration across analog and digital blocks: connecting the front-end, memory elements, and serialization flow.
